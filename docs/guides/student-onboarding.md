# Student Onboarding Guide

This guide helps teams onboard students with different programming backgrounds.

## Skill tracks

### Track A - Beginner (Week 1)

- Install toolchain and run robot simulation.
- Read `robot.py` and `robot_container.py`.
- Complete one safe change: add a controller binding.

### Track B - Intermediate (Week 2-3)

- Implement one subsystem using the IO pattern.
- Add basic state transitions and command factory methods.
- Add a **superstructure goal** that sets your subsystem together with another mechanism.
- Add one unit-style test or sim validation checklist.

### Track C - Advanced (Week 3-4)

- Lead one architecture area (auto, vision, or superstructure).
- Review PRs from Track A/B students.
- Maintain one documentation section.

## Definition of done (all tracks)

- Code compiles and runs in sim.
- Change is documented in docs.
- At least one mentor and one student review the PR.
- Driver impact is called out clearly.
