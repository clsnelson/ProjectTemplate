# R303 Compliance Checklist (Software Reuse)

This checklist helps keep yearly reuse legal and transparent.

## Relevant rules to verify each season

- R303 (pre-Kickoff software/designs must be public if reused)
- R302 (major mechanisms built in-season)
- R301 (software/item FMV constraints)

Always confirm current season Team Updates and Q&A.

## Team process

1. Keep template and shared library source public before Kickoff.
2. Tag release versions before Kickoff.
3. In the season repo, record links to those public tags.
4. Record which reused modules are game-agnostic.
5. Avoid carrying private prebuilt game-specific code into season repo.

## Evidence to keep

- Public release tag links
- Commit hashes used in season repo
- Mentor sign-off note for compliance check
- Date of last Team Update/Q&A review

## Escalation

If unclear whether a reused module is game-specific, ask on official FRC Q&A before relying on it in competition robot software.
