# New Season Checklist

Use this checklist when creating a new season repository from this template.

## Before Kickoff

1. Tag and publish this template with latest stable changes.
2. Ensure reusable code and docs are public.
3. Confirm shared-library versions are released and documented.

## Kickoff Week

1. Create season repository from this template.
2. Update robot name, team info, and season metadata.
3. Update `robot_config.py` with current practice/comp robot maps.
4. Add game-specific constants placeholders in `constants.py`.
5. Stand up minimal drivetrain + controller control.

## Week 1-2

1. Implement base subsystem scaffolds for planned mechanisms.
2. Add initial autos and test in simulation.
3. Create first student issues by skill level.

## Every PR

1. Verify tests/sim still run.
2. Confirm docs and controller bindings are updated.
3. Check if any change affects FRC compliance checklist.
